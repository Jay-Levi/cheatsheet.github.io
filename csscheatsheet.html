Position -MDN-

Static

Static is the default for the positon property. without defining the position of an element, it is positioned according to the normal flow of an HTML element.

Relative

The relative value allows an element to be positioned relative to where it would have originally been on a web page. Its position on the web page can be offset using the 'top' 'bottom' 'left' or 'right' properties. without an offset the declaration will have no different on the element. (Note how that there is still a space saved for the element between the boxes.)

Absolute

This declaration removes the element from the normal flow of the document and all other elements act as if it is not present on the web page. its position can be offset using the 'top' 'bottom' 'left' or 'right' properties. (Note that there is no space left for the element in this section.)

Fixed

This declaration fixes an element to a specific position on the page (regardless of user scrolling). its position can be offset using the 'top' 'bottom' 'left' or 'right' properties. (there is no example for this declaration on this page. It would make it hard to read!) Click here to view fixed positioning.

Sticky

This declaration is positioned according to the normal flow of the document but it's position 'sticks' to its nearest ancestor that has a scrolling mechanism.

Skeuomorphism & Flat Design

Skeuomorphic
Flat

Skeuomorphic

Skeuomorphism is a design concept aimed to make items on the web or on mobile, resemble items in the real world. The button in the example is designed to resemble a real life button using CSS properties.

Flat design

Flat design is a more of a simplistic design concept. In UX design, developers must rely on styling and signifiers to indicate what it is they are trying to create.
